# 평가 루브릭

## 점수 항목 (총 100점)

평가는 다음 4개 항목으로 구성되며, 각 항목별 배점은 다음과 같습니다:

1. **Discoverability (30점)** - 사용자가 스킬을 쉽게 찾고 이해할 수 있는가?
2. **Clarity (30점)** - 스킬 정의가 명확하고 일관성 있는가?
3. **Completeness (20점)** - 스킬이 필요한 모든 요소를 포함하는가?
4. **Testability (20점)** - 스킬의 동작을 검증할 수 있는가?

---

## 1. Discoverability (30점)

**목적**: 사용자가 스킬을 쉽게 찾고, 무엇을 하는지 빠르게 이해할 수 있어야 합니다.

### 세부 기준

| 항목 | 배점 | 체크리스트 |
|------|------|------------|
| **명확한 이름** | 10점 | - 스킬 이름이 기능을 직관적으로 설명하는가?<br>- 네이밍 컨벤션(kebab-case, 접두사 등)을 따르는가?<br>- 중복되거나 혼동되는 이름이 아닌가? |
| **간결한 설명** | 10점 | - description이 1-2문장으로 핵심을 요약하는가?<br>- 사용 시나리오가 명확한가?<br>- 기술 용어를 최소화했는가? |
| **적절한 메타데이터** | 10점 | - frontmatter가 모든 필수 필드를 포함하는가?<br>- tags가 적절하게 분류되었는가?<br>- user-invocable이 올바르게 설정되었는가? |

---

## 2. Clarity (30점)

**목적**: 스킬의 동작 방식과 사용법이 명확하고 일관되어야 합니다.

### 세부 기준

| 항목 | 배점 | 체크리스트 |
|------|------|------------|
| **명확한 입력/출력** | 10점 | - 필요한 인자가 명시되어 있는가?<br>- 각 인자의 타입과 의미가 설명되었는가?<br>- 출력 형식이 정의되어 있는가? |
| **구체적인 예시** | 10점 | - 실제 사용 예시가 포함되어 있는가?<br>- 예시가 다양한 시나리오를 커버하는가?<br>- 예시 코드가 실행 가능한가? |
| **일관된 구조** | 10점 | - 문서 구조가 표준 템플릿을 따르는가?<br>- 섹션 순서가 논리적인가?<br>- 마크다운 문법이 일관되는가? |

---

## 3. Completeness (20점)

**목적**: 스킬이 실제로 동작하는 데 필요한 모든 정보를 포함해야 합니다.

### 세부 기준

| 항목 | 배점 | 체크리스트 |
|------|------|------------|
| **필수 정보 포함** | 10점 | - 모든 frontmatter 필드가 채워져 있는가?<br>- 실행 조건(dependencies, permissions)이 명시되었는가?<br>- 에러 처리 방법이 설명되어 있는가? |
| **컨텍스트 정보** | 10점 | - 언제 사용해야 하는지 설명되었는가?<br>- 다른 스킬과의 관계가 명시되었는가?<br>- 제약사항이나 주의사항이 기술되었는가? |

---

## 4. Testability (20점)

**목적**: 스킬의 동작을 검증하고 품질을 보장할 수 있어야 합니다.

### 세부 기준

| 항목 | 배점 | 체크리스트 |
|------|------|------------|
| **검증 가능한 명세** | 10점 | - 성공 조건이 명확하게 정의되었는가?<br>- 실패 케이스가 식별되었는가?<br>- 예상 결과가 구체적으로 기술되었는가? |
| **테스트 지침** | 10점 | - 수동 테스트 방법이 제시되었는가?<br>- 자동화 가능한 검증 포인트가 있는가?<br>- 회귀 테스트 고려사항이 명시되었는가? |

## 사용 가이드

### 평가 수행 방법

1. **각 항목별 독립 평가**: 4개 항목을 개별적으로 평가
2. **체크리스트 확인**: 각 항목의 세부 체크리스트 검토
3. **점수 산정**: 체크리스트 통과 비율에 따라 배점 부여
4. **총점 계산**: 4개 항목 점수 합산

### 복수 평가 프로토콜

- **측정 횟수**: 3회 반복 측정
- **독립성**: 각 측정은 독립적으로 수행
- **신뢰구간**: 95% 신뢰구간 계산
- **유의미성**: 신뢰구간 분리 여부로 판단
